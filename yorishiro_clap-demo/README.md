# yorishiro_clap-demo

Pregel cell for the **clap-demo** yorishiro (kami: `bin:clap-demo`,
binary: `clap-demo`).

Per ADR-2605211900 + ADR-2605202200. Generated by `@etzhayyim/yorishiro`
v0.1.0 in **binary-cli** mode.

## Ops

| Op | Summary |
|---|---|
| `clap-demo` | Demo clap CLI used by the yorishiro source-repo fixture. |

## Runtime requirements

The `clap-demo` binary must be available on the
cell runtime's PATH (or supplied as an absolute path in the kami
manifest at `00-contracts/kami/clap-demo.kami.json`). The cell
verifies this via `shutil.which()` at the start of every invocation
and returns `error: "binary not found on PATH"` if missing.

## Lexicon SSoT

`00-contracts/lexicons/ai/etzhayyim/yorishiro/clap-demo/<op>.json`

## MCP exposure

`kotoba-lang/kotodama-mcp/yorishiro-clap-demo-mcp/` (stdio + Streamable HTTP)

```bash
yorishiro regen clap-demo
```

## Claude Desktop / Codex CLI

```json
{
  "mcpServers": {
    "etzhayyim-yorishiro-clap-demo": {
      "command": "node",
      "args": ["/ABSOLUTE/PATH/TO/repo/kotoba-lang/kotodama-mcp/yorishiro-clap-demo-mcp/src/cli.ts"]
    }
  }
}
```

The binary itself does NOT need to be on the MCP host machine if you
intend the MCP server to dispatch to a remote runtime. For the simple
local-binary case shown above, `clap-demo` must be
on the host's PATH.
