# yorishiro_pdftotext

Pregel cell for the **pdftotext** yorishiro (kami: `bin:pdftotext`,
binary: `pdftotext`).

Per ADR-2605211900 + ADR-2605202200. Generated by `@etzhayyim/yorishiro`
v0.1.0 in **binary-cli** mode.

## Ops

| Op | Summary |
|---|---|
| `convert` | Convert a PDF file to plain text |

## Runtime requirements

The `pdftotext` binary must be available on the
cell runtime's PATH (or supplied as an absolute path in the kami
manifest at `00-contracts/kami/pdftotext.kami.json`). The cell
verifies this via `shutil.which()` at the start of every invocation
and returns `error: "binary not found on PATH"` if missing.

## Lexicon SSoT


```json
{
  "mcpServers": {
    "etzhayyim-yorishiro-pdftotext": {
      "command": "node",
      "args": ["/ABSOLUTE/PATH/TO/repo/20-actors/kotoba-kotodama/mcp/yorishiro-pdftotext-mcp/src/cli.ts"]
    }
  }
}
```

The binary itself does NOT need to be on the MCP host machine if you
intend the MCP server to dispatch to a remote runtime. For the simple
local-binary case shown above, `pdftotext` must be
on the host's PATH.
