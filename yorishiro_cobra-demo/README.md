# yorishiro_cobra-demo

Pregel cell for the **cobra-demo** yorishiro (kami: `bin:cobra-demo`,
binary: `cobra-demo`).

Per ADR-2605211900 + ADR-2605202200. Generated by `@etzhayyim/yorishiro`
v0.1.0 in **binary-cli** mode.

## Ops

| Op | Summary |
|---|---|
| `cobra-demo` | Demo cobra CLI used by the yorishiro source-repo fixture. |
| `greet` | Print a greeting. |
| `render` | Render output to file or stdout. |

## Runtime requirements

The `cobra-demo` binary must be available on the
cell runtime's PATH (or supplied as an absolute path in the kami
manifest at `00-contracts/kami/cobra-demo.kami.json`). The cell
verifies this via `shutil.which()` at the start of every invocation
and returns `error: "binary not found on PATH"` if missing.

```json
{
  "mcpServers": {
    "etzhayyim-yorishiro-cobra-demo": {
      "command": "node",
      "args": ["/ABSOLUTE/PATH/TO/repo/kotoba-lang/kotodama-mcp/yorishiro-cobra-demo-mcp/src/cli.ts"]

```

The binary itself does NOT need to be on the MCP host machine if you
intend the MCP server to dispatch to a remote runtime. For the simple
local-binary case shown above, `cobra-demo` must be
on the host's PATH.
